## 3. КАК СОБИРАЕТСЯ СТАТИСТИКА С КАНАЛОВ — ТЕКУЩЕЕ VS АВТОМАТИЗАЦИЯ

### Текущее состояние: 100% ручной ввод
Сейчас пользователь сам вбивает каждую неделю:
- Shorts/Long-form опубликовано
- Средние просмотры, удержание %
- Прирост подписчиков по 5 платформам
- Engagement (комменты, DM)
- Часы работы, инсайты

### Как автоматизировать:

#### YouTube Data API v3 (бесплатно, 10,000 запросов/день)
```
GET https://www.googleapis.com/youtube/v3/channels?part=statistics&id={CHANNEL_ID}
→ subscriberCount, viewCount, videoCount

GET https://www.googleapis.com/youtube/v3/videos?part=statistics&id={VIDEO_ID}
→ viewCount, likeCount, commentCount
```
- **Нужно**: Google Cloud проект → YouTube Data API → OAuth или API Key
- **Даёт**: подписчики, просмотры, лайки, комменты, количество видео
- **Не даёт**: удержание (требует YouTube Analytics API + OAuth)

#### TikTok Research API
- Требует Business аккаунт + подача заявки
- Альтернатива: парсить через TikTok Display API (ограничено)

#### Instagram Graph API
- Требует Business/Creator аккаунт + Facebook App
- Даёт: followers, media_count, impressions

#### Telegram Bot API
- Создать бота → добавить в канал → `getChat` → member_count
- Самый простой из всех

#### X (Twitter) API v2
- Free tier: 1,500 tweets/месяц, read only
- `GET /2/users/{id}` → followers_count

### Реалистичный план автоматизации:
1. **YouTube** — подключить первым (самый важный, бесплатный API)
2. **Telegram** — второй (самый простой)
3. **TikTok/Instagram/X** — позже (требуют бизнес-аккаунты)

## 5. АРХИТЕКТУРНЫЕ РЕКОМЕНДАЦИИ

### Текущая проблема: всё в localStorage
- **Риск**: один `localStorage.clear()` = потеря всех данных
- **Решение**: добавить Export/Import + опционально Supabase/Firebase для бэкапа

### API ключи в теле запросов
- **Текущее**: ключи передаются в JSON body от клиента к серверу
- **Лучше**: `.env.local` для серверных ключей, без передачи через сеть
- Но для multi-key (user может менять) текущий подход допустим

### Нет тестов
- Нет unit/integration тестов
- Рекомендация: хотя бы smoke-тесты для API routes
